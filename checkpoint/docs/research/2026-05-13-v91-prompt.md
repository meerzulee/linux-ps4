## Still observed in v89 boot

```
BAR+0x04 linkreg = 0x00000b18 (bit 0 = 0, but speed/duplex fields populated)
BAR+0x208 = 0x00000001 (auto-set, never written by software)
BAR+0x210 = 0x00000001 (auto-set, never written by software)
BAR+0x118 = 0x000000a7 (auto-incrementing counter — MAC IS receiving symbols)
BAR+0x128 = 0x00000080
BAR+0x12c = 0x00000096
```

Writing `BAR+0x09c = 0xff` flips `linkreg` speed field from 1000M→100M
indicators (proven MAC reflects PHY state changes, just not bit 0).
