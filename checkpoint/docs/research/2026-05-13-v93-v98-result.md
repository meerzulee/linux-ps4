# v93→v98 result — Phase 3 netdev + RX working, TX dead (2026-05-13)

## TL;DR

After v91's link UP, the goal was to wrap the working phase-1 PHY/MAC
bring-up in a real netdev (mts0/enp0s20f1).  Five iterations later:

- ✅ **Netdev registered** (v93) — `ip link show enp0s20f1` works
- ✅ **Carrier UP** (v94/v95) — PHY-derived signaling bypasses unreliable
  MAC bit 0 latch
- ✅ **RX FULLY WORKING** (v97) — paired-register fix made HW read RX
  descriptors; 12+ packets delivered via NAPI to netdev counters
- ❌ **TX completely dead** — descriptors correctly formatted in DMA
  memory but HW never sets OWN=1, no TX_DONE IRQ, host receives nothing

## TX-dead evidence (v98 boot)

`v98 reap probe` after 47 xmits with bit 18 IRQs flooding:

```
v98 reap probe: tx_prod=47 tx_cons=0 tx_ring[0] ctl=0x30000149 a0=0x00000000
```

- `ctl=0x30000149` is our submit (SOF | EOF | length=329, OWN=0)
- HW has not flipped bit 31 (OWN) to indicate completion
- `tx_ring[0]` reads identical to what driver wrote

Descriptor format VERIFIED correct via v98 dumps for entries 21..33:
- ctl always `0x30000000 | length` (SOF + EOF + length)
- buf_lo always a real DMA address (0x7b61e000, 0x7c6fc000, etc.)
- aux0 = 0 (sentinel cleared)
- aux1 = 0

## Live BAR-poke session (no reboot, on v95/v96/v97)

State observed:
- `BAR+0x034` (TX_CTRL): bits 0+2 R/W only; bit 2 (kick) sticks once
  set (doesn't auto-clear like a pulse register would)
- `BAR+0x038` (RX_CTRL): write bit 0 → readback shows bit 3 (running)
- `BAR+0x03c` (TX ring base): stationary at tx_ring_dma, never advances
- `BAR+0x040` (RX cursor): advances by 0x10 per RX packet (HW pointer)
- `BAR+0x044` paired: live test confirmed should be `tx_ring_dma` not
  `upper_32_bits` (= 0 with 32-bit DMA mask) — fix in v97 made RX work

Attempted live unsticking (none worked):
- `0x034 |= 4` kick (multiple times, fresh sequences)
- 0x034 reset cycle: w(0), w(1), w(5)
- All kick values 0x4/0x6/0x7/0x101/0x104/0x47/0x80000005/0xff
- 0x09c bit 6 toggle: DESTRUCTIVE (zeroes BAR+0x040, breaks RX too).
  hermes confirmed this is TX-error recovery only.
- 0x044 = 1 (TX_PROD attempt): no effect
- MAC_ADDR0 program (0x014/0x018 = netdev MAC): no effect
- 0x078 RX_GATE toggle: no effect
- 0x038 bit 0 + bit 2 in various sequences: no effect
